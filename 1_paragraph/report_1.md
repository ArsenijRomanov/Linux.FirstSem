## Ход работы

## 1. Базовые команды

### 1.1 Вывести все дочерние директории в `/proc`, игнорируя ошибки доступа
```sh
find /proc -mindepth 1 -type d 2>/dev/null
```

**Пояснение:**
- `find /proc` — обход каталога `/proc`.
- `-mindepth 1` — не печатать сам `/proc`, только вложенные элементы.
- `-type d` — выводить только директории.
- `2>/dev/null` — скрыть сообщения об ошибках доступа

---

### 1.2 Вывести строки файла в обратном порядке
```sh
tac 1_1.sh
```

**Пояснение:**
- `tac` выводит строки файла **с конца к началу** (reverse `cat`).

---

### 1.3 Показать список файлов текущей директории с правами (включая скрытые), директории — первыми
```sh
ls -lA --group-directories-first
```

**Пояснение:**
- `-l` — подробный список, включая права доступа, владельца, группу, размер и дату изменения
- `-A` — показать скрытые файлы, но не выводить `.` и `..`.
- `--group-directories-first` — выводить директории перед файлами.

---

## 2. Файловая система sysfs

### 2.1 Считать параметр из `sysfs` (температура CPU / заряд батареи)
```sh
# Температура CPU 
awk '{printf("%.1f°C\n",$1/1000)}' /sys/class/thermal/thermal_zone0/temp

# Уровень заряда батареи (0–100)
cat /sys/class/power_supply/BAT1/capacity
```

**Пояснение:**
- `sysfs` (`/sys`) — интерфейс ядра к устройствам и их параметрам.
- awk читает вход построчно и делит каждую строку на поля, разделённые пробельными символами
- Температура в `thermal_zone*/temp` часто хранится в **миллиградусах** → делим на 1000.
- Заряд батареи доступен в `BAT*/capacity` в процентах.
